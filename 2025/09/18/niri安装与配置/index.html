前言

之前我的 Arch 用的一直是 KDE Plasma，也在 Debian 上用过 xfce，但是 KDE 采用的是仿 Windows 的艺术风格，虽然也可以折腾得很美观，但是内存占用很大，开机占用基本保持在 3G 左右；xfce 作为一个最小可用的桌面环境，感觉有些过于精简了，系统设置的图形界面也不太好用。

前两天冲浪的时候偶然看到有人展示自己的 niri 桌面环境，还有人说自己入了 niri 的坑之后马上抛弃 Hyprland 了，理由是 niri 在美观程度不输 Hyprland 的同时，提供了很多【开箱即用】的体验，用户需要自主配置的难度大大降低。于是我花了一整天的时间在 VMware 虚拟机里折腾 niri，最后配置出来了比较完整的桌面环境，等以后实习攒够钱组了台式机之后就按照这个流程装桌面环境好了。

配置完毕的效果如下：

既比 KDE 轻量，也要美观得多。开机之后的内存占用基本上不会超过 1G，这大概是得益于 Rust 的高效率？（顺便一提，Hyperland 是基于 C++ 写的，现在大批用户涌入 niri 间接让 Rust 又小赢了一波😋）

niri 的官网在这里，还是挂在作者的 Github Pages 上的一个小网页，但是基本上把该讲的东西都讲清楚了，建议好好读一下。

基本安装

先安装一个ArchLinux系统，可以参考Archlinux简明指南，不再赘述。

如果是在 VMware 安装 Archlinux，参考：https://www.cnblogs.com/Thato/p/18311683，关键点只有一个，就是在开始安装之前需要手动修改配置文件，确保启动方式是 UEFI。剩下的参考ArchLinux安装指南即可。

系统安装完毕后，使用下面的命令安装必要的包：

  • niri：平铺式窗口管理器
  • alacritty：GPU加速的终端模拟器
  • fuzzel：用于快速搜索应用
  • swaylock：搭配swayidle，实现闲置一段时间后锁屏
  • swayidle：闲置管理工具，可以设置超时后执行的动作
  • waybar：高度可自定义的 Wayland 状态栏
  • swaybg：Wayland 的桌面壁纸管理工具
  • xwayland-satellite：用于运行 x11 应用的 Wayland 桥接工具
  • gdm：登录管理器
  • mako：通知管理器

如何修改系统配置

如果希望只修改当前用户的配置，应该修改~/.config/niri/config.kdl。/etc/niri/config.kdl的修改适用于所有用户。

配置文件的修改会在保存之后立刻生效。如果配置有误，可以调用niri validate查看详情。

锁屏设置

swaylock的相关命令见官方文档

swaylock不会自动创建配置文件，而是先去按照优先级查找：

  1. $HOME/.swaylock/config
  2. $XDG_CONFIG_HOME/swaylock/config（通常是 $HOME/.config/swaylock/config）
  3. ·SYSCONFDIR/swaylock/config（在 Arch 上就是 /etc/swaylock/config）

而且，原版 swaylock 支持的效果是非常少的，想要使用 clock、blur 之类的特效的话需要安装插件：https://github.com/mortie/swaylock-effects，但是这个项目已经4年没有更新了，基本没法用，所以先用着原版吧，虽然简陋一点。
