Rust模块系统知识梳理

Rust模块系统

总览

Rust的模块系统包括：

  • Packages: Cargo的一个功能，允许你 build test share crate
  • Crates： 单元包/程序包，由一棵模块树组成，可以生成库（library）或可执行文件
  • Modules and use：模块和use关键字，可以控制路径的组织方式，作用域和可见性
  • Paths：一种命名事物的方式，例如结构体，函数或者模块

crate的分类：

  • 二进制crate,这种crate可以被编译成可执行文件，必须包含main函数。
  • Library crate：不能被编译成可执行文件，不包含main函数，它们主要用来为不同的项目提供可用的功能函数。大部分情况下，我们所说的crate指的是这个，而且它的含义接近一般的编程语言中的“库”的概念。

package是由多个crate组成的，一个package包含一个Cargo.toml文件用来描述如何构建这些crate,可以认为一个package就是一个包含toml的项目目录。包括Cargo本身也是一个package,它包含了很多命令行二进制文件。

  • 一个package最多只能有一个library crate，可以有多个二进制crate
  • 一个package至少包含一个crate

也就是说，一个package可以有任意多个可执行文件，但是只能有一个库。

Cargo 遵循一个约定：

  • src/main.rs 是与包同名的二进制 crate 的 crate root。

同样，Cargo 知道，如果包目录包含 src/lib.rs，则该package包含一个与包同名的lib crate，而 src/lib.rs 是它的 crate root。Cargo 将 crate 根文件传递给 rustc 来构建库或二进制文件。

定义模块

路径用于定义组件的名字，可以使用use导入路径。

编译器的处理步骤：

  • 从 crate root 开始查找要编译的代码，比如src/main.rs
  • 声明模块：在crate root 可以声明模块，比如mod foo，然后编译器会在下面的地方寻找模块的代码：
    • inline： 直接在{}写代码
    • 单独的文件：把模块的名字当成文件名，/src/foo.rs
    • 目录文件：把模块的名字当成目录名，/src/foo/mod.rs
  • 声明子模块：在任何非crate root 文件里都可以声明子模块，比如/src/foo.rs里面写mod bar，则编译器会在下面三个地方找：
    • inline
    • 单独的文件/src/foo/bar.rs
    • 目录：/src/foo/bar/mod.rs
  • 一旦在crate中声明了模块，那么就可以在同一个crate的其他地方使用module的代码了。引用路径即可。例如：crate::foo::bar::类型名，即foo模块中的bar子模块包含的类型。
  • 可见性：默认情况下，模块内的代码是私有的，父模块不能调用。pub mod声明模块，可以让模块变成可见的。模块内的类型、函数等也需要pub才能变得可见。
  • use关键字：在一个作用域内，可以使用 use 关键字为路径创建快捷方式，从而避免重复输入长路径。例如，use crate::foo::bar::my_type，然后就可以直接用my_type引用这个类型了。

这里再补充一句，既然使用pub mod 模块名才能使得模块变得可见，那么只写mod xxx的话岂不是无法引用xxx模块的任何项？也就是说mod xxx是没有意义的？并非如此，因为mode xxx的主要目的是告诉编译器把 xxx 这个模块包含进当前 crate 的编译过程，把这个模块编译进来，这是一个声明的过程，而不是访问控制，不能访问这个模块中的项。如果想要访问模块中的代码，才需要pub。

子模块可以看到父模块里面定义的函数，方法，结构等，但是父模块只能看到声明为pub的子模块。这样的好处是，子模块的内部实现细节默认是被隐藏起来的，换句话说这一类模块进行内部代码更改的时候不会对外界产生影响只有标记为pub的才会产生影响。

同时，定义在统一层级的兄弟模块是可以直接互相访问的，不需要pub。访问子模块才需要pub

建议：在一个项目中同时包含/src/lib.rs /src/main.rs，前者专门用来定义模块树，后者可以被简单视作库crate的一个调用者，就和外部调用者一样，只允许调用pub的API.你不仅是作者，也是自己的客户！

use关键字

使用use把模块路径/函数路径等直接引入use所在的作用域，类似于文明系统中创建符号链接，是一个快捷方式。

约定俗成的习惯： 如果要把一个函数引入作用域，最好use函数所在的父模块，而不是直接use函数本身的路径，通过父模块调用函数，显得更加清晰，表明这个函数是从外部引入的，而不是定义在当前工程的。

相反，在导入结构、枚举之类的会直接导入全路径，这样也方便区分函数和其他结构。
